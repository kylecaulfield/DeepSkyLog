iOS share-sheet shortcut

Install once on your iPhone, then "Share → DeepSkyLog" from the Seestar app (or anywhere else that exports an image) creates an observation here in one tap.

Why this isn't a one-tap install

Apple requires shortcuts to be signed on a Mac before they can install without warnings. Until you build a signed version once and host it on iCloud, the practical path is the recipe below — about five minutes on your phone. Once it's built, every future upload is one tap.

Before you start

  1. On your iPhone, open Settings → Shortcuts and turn on Allow Untrusted Shortcuts. (You only need this if you ever want to import a `.shortcut` file from outside the App Store — building one by hand doesn't require it, but it's good to flip on anyway.)
  2. Open the Shortcuts app and tap the + in the top-right to start a new shortcut. Name it DeepSkyLog.
  3. Tap the shortcut's (i) info icon and turn on Show in Share Sheet. Set Share Sheet Types to Images only.

The actions

  1. Receive [Images] input from Share Sheet — already in place from the toggle above.
  2. Get Contents of URL — first call: stage the file.
    URL …
    Set Method to POST.
    Tap Headers, leave empty.
    Under Advanced, switch Request Body to Form. Add one entry:
    • Key: image
    • Type: File
    • Value: Shortcut Input (variable)
    Open Advanced → Authentication, choose Basic, set User to admin and Password to the same ADMIN_PASSWORD you use to log in here.
  3. Get Dictionary Value — pull stage_id out of the response.
    Key stage_id
    Set the dictionary to the Contents of URL variable from the previous step.
  4. Set Variable — save the dictionary value as a variable named stage. Makes the next action cleaner.
  5. Ask for Input (optional) — prompt for the target name. Question: What is this an image of?, Input Type: Text, Default Answer: (leave blank). Save as variable target. Skip this step if you'd rather always log as free-form and edit the target later in the web admin.
  6. Get Contents of URL — second call: finalize the observation.
    URL …
    Method: POST.
    Headers: add one — Content-Type: application/json.
    Request Body: JSON. Build the object:
    • stage_id → variable stage
    • object_name → variable target (or skip if you didn't add the prompt)
    • telescope → Seestar S50 (set to whatever you image with)
    • observed_at → Current Date, format ISO 8601
    Repeat the same Basic auth setup as in step 2 under Advanced.
  7. Get Dictionary Value — read the new observation id. Key: id, Dictionary: Contents of URL from the previous step. (Optional but it makes the notification informative.)
  8. Show Notification — body: Saved observation #Dictionary Value using the id from the previous step. Title: DeepSkyLog.

Test it

  1. Tap the play button at the top of the shortcut. It'll prompt you to pick a photo. Pick anything. You should see a notification with the new observation number, and the image should appear under Observations.
  2. Now share an image from your Seestar app (or from Photos → Share). The "DeepSkyLog" entry should appear in the share sheet alongside Messages, Mail, etc.

Tweaks

  • No prompt at all. Skip step 5 and drop object_name from step 6's JSON. The shortcut becomes one-tap and you edit the target on the web later.
  • Multiple images. Wrap steps 2–8 in a Repeat with Each action over Shortcut Input so dragging multiple images at once stages each one.
  • Different telescope per session. Replace the hardcoded telescope in step 6's JSON with a Choose from List action listing your scopes.
